005.谈下python的GIL

gil是python的全局解释器锁，他并不是python的特性，而是cpython解释器的特性（我们平时使用的都是cpython）。
gil是一个全局锁，所有的python都给它上了锁，执行前获取gil锁，执行完毕释放锁让其他线程执行。
因为gil的存在所以同一时刻在一个cpu上只能运行一个线程，无法将多格线程映射到多个cpu上，所以python里的多线程实际上是伪多线程，如果要利用多核cpu的话就要用多进程（但是系统资源开销大）。
但在遇到io操作时也会释放gil锁，所以在io密集型程序用多线程也会提高效率，而遇到计算密集型的程序因为切换线程的开销不但不会提高效率反而可能有副作用。

007.fun(*args,**kwargs)中的*args,**kwargs什么意思？

分开四部分讲：*，*args，**，**kwargs
def fun(a,b,c):
    print(a,b,c)

1.我们调用fun的时候不能fun([1,2,3])，这样会报错，但如果fun(*[1,2,3])就可以，*就是将后面的序列类型拆开一个个的传入函数。

2.有时候我们并不知道有多少个参数传入函数，这时候我们就可以用*args，fun(*args)，args会把传入的参数打包成一个tuple，*args就是把这个tuple逐个传入函数。它只适用于非键值对类型，dict就不行。

3.**和*类似，也是拆包，不过它拆的是dict，调用fun(a,b,c)时我们可以直接fun(1，2，3)，也可以写成fun(a=1,b=2,c=3)，但是我们不能传入fun({a:1,b:2,c:3})，但是有了**就可以，fun(**{a:1,b:2,c:3})，它会将后面的dict拆成键值对的形式传入函数。

4.我们不知道要把什么参数传进函数的时候就可以用**kwargs，fun(**kwargs)，fun(a=1,b=2,c=3)，kwargs会把参数列表打包成一个dict，我们在函数内kwargs[a]就可以获取a的值。

5.总结一下就是，当我们不知道函数内要传入多少个值/传什么值的时候就可以使用*args,**kwargs这种万能参数

011.简述面向对象中__new__和__init__区别

class Person(object):
    def __new__(cls, name, age):
        return super(Person, cls).__new__(cls, name, age)
    def __init__(self, name, age):
        self.name = name
        self.age = age

在new一个对象的时候，先执行__new__，生成这个实例，然后再执行__init__，初始化这个实例，比如给它添加属性，
__init__里面传入的self实际上就是__new__生成的实例。
所以__new__必须要有返回值，通常调用父类的__new__方法，return super(类名,cls)__new__
而__init__不需要返回值

012.简述with方法打开处理文件帮我我们做了什么？

使用with可以简化代码，它实际上就是调用了两个魔法函数，__enter__和__exit__，
一般在__enter__里获取资源，在__exit__里释放资源，比如说io操作结束的时候我们一般要写close()，
有了with我们就可以把close()定义在__exit__里面，就不怕忘写close了
我们自己写的类如果实现了这两个魔法函数也可以使用with

016.<div class=”nam”>中国</div>，用正则匹配出标签里面的内容（“中国”），其中class的类名是不确定的

import re
s = '<div class="nam">中国</div>'
result = re.findall(r'<div class=".*">(.*?)</div>', s)
print(result)
# .*提取文本，(.*?)提取文本

re.findall()    匹配所有的，返回一个list
re.search()        只返回匹配的第一个，返回一个匹配对象，匹配不到返回None
re.match()        和search差不多，只不过是从起始位置匹配，不是起始位置也返回None
re.compile()    生成正则对象，供search和match使用

正则判断一个字符串是否为浮点数，如：
s = ‘5.0129’
pattern = re.compile(r'^[-+]?[0-9]+\.[0-9]+$')
# pattern = re.compile(r'^-?\d+\.\d+$')
result = pattern.match(s)
if result:
    print('这是浮点数')
else:
    print('这不是浮点数')

关于这个正则表达式，解释一下：
^[-+]?[0-9]+\.[0-9]+$     # \d同[0-9]
^表示以这个字符开头，也就是以[-+]开头，[-+]表示字符-或者+之一        #一般正数不加+，所以可以省略[-+]中的+ 
?表示0个或1个，也就是说符号是可选的。
同理[0-9]表示0到9的一个数字，+表示1个或多个，也就是整数部分。
\.表示的是小数点，\是转义字符因为.是特殊符号（匹配任意单个除\r\n之外的字符），所以需要转义。
小数部分同理，$表示字符串以此结尾。

2.使用__new__

class Singleton(object):
    _instance = None
    def __new__(cls, *args, **kwargs):
        if not _instance:
            cls._instance = super(cls, *args, **kwargs)
        return cls._instance

class Myclass(Singleton):
    pass

3.使用装饰器

def singleton(cls):
    instances = {}
    def getinstance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    return getinstance

@singleton
class MyClass(object):
    pass

4.使用元类metaclass

class Singleton(type):
    _instance = {}
    def __call__(cls, *args, **kwargs):
        if cls not in cls._instance:
            cls._instance[cls] = super(Singleton, cls).__call__(*args, **kwargs)
        return cls._instance[cls]

class MyClass(metaclass=Singleton):
    pass

062.简述cookie和session

cookie：
    http协议是无状态协议，一旦数据交换完毕它的连接就关闭了，再次交换数据需要建立新的连接，两次连接之间是没有联系的。
    如果要记录一个用户在某网站上的行为，就要建立会话跟踪机制，cookie就应运而生了，本质上cookie就是http协议的一个拓展。
    你的原理就是你在初次访问某网站的时候它会给你一个颁发一个号牌
    （这个号牌就是cookie，放在http的头部set-cookie里，存在你的浏览器中），
    以后你每次发请求时都带着这个号牌，服务器就知道这是你发送的请求了。
    cookie有时间限制，会过期。而且由于它存在客户端里，所以安全性较差。
session：
    Session是另一种记录客户状态的机制，不同的是Cookie保存在客户端，而Session保存在服务器上。
    如果说Cookie是通过检查客户身上的“通行证”来确定客户身份的话，那么Session相当于程序在服务器上建立的一份客户档案，客户来访的时候只需要查询客户档案表就可以了。
    用户访问服务器的时候，服务器把客户端信息记录在服务器上，这就是Session，客户端浏览器再次访问时只需要从该Session中查找该客户的状态就可以了。
    session的运行依赖于cookie（session id是存在cookie中的，有它才可以在客户档案表中查到信息），如果浏览器禁用cookie那么session也将失效。
    session也有有效期，这个有效期时服务器设定的，如果用户没有访问网站的时间超过了有效期，该session就会过期失效。

063.简述多线程、多进程

多进程：
    操作系统进行资源分配和调度的基本单位，各进程之间相互独立，稳定性好，一个进程奔溃了不影响其他进程，但是进程资源消耗大，开启的数量有限制。
    多进程可以充分利用多核CPU
多线程：
    线程是进程的一部分，它是CPU进行资源分配调度的基本单位，一个进程下的多个线程可以共享该进程的所有资源，
    所以如果多个线程要修改同一个对象时就要加锁，如果操作不当就会造成死锁。
    因为cpython中GIL的存在，多线程无法利用多核CPU
应用：
    IO密集的用多线程，它在等待IO时会切换到其他线程执行。但是进程间的切换代价要大于线程。
    计算密集的用多进程，如果用多线程的话没法充分利用多核CPU，并且切换线程的开销不但不会提高效率还可能会有副作用

066.python中copy和deepcopy区别

浅拷贝和深拷贝的不同就在于复制的对象中有可变对象的情况，如一个list中还有一个list，
如果我们改变外层list的值，浅拷贝的对象不会改变，但如果改变内层list里的值，浅拷贝的对象也会发生变化，
也就是说，它只拷贝了一层；
而深拷贝就不会，它是把对象整个复制了一份。
